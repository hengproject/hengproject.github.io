Data preprocessing

Why do we need preprocess

  • Activation functions (ReLU) works around zero

If input data is biased toward positive or negative, randomly initialized layers produce biased output.

  • A feature with small scale has a negligible effect on backprop.

  • Remove correlation between input features (PCA Whitening)

And a lot more …

Normalizations

Normalization : Normalize the distribution of activations to have zero mean and unit variance

Why we shall do normalization:

  • Internal Covariate Shift

provides the theoretical basis for why normalization is necessary—it stabilizes the input distribution to layers and speeds up learning.

  • To Train Deeper Networks on Larger Data

For deeper networks and larger datasets, normalization improves training stability, enhances convergence speed, and ensures better performance.

Batch Normalization


[Ioffe and Szegedy, 2015]

Why it is good?

  • Improves gradient flow
  • Allows higher learning rates
  • Reduces strong dependence on initialization
  • Regularization

Usually added after convolutional layers or fully connected layers && before non-linearities

Missing Datapoint

Missing data can be handled in several ways:

  1. Imputation: Replace missing values with a statistic like mean, median, or
    mode, or use a model to predict missing values.
  2. Removal: If the missing values are not random or comprise a large portion
    of the data, it might be better to remove those points.
  3. Indicator Variable: For some algorithms, it can be useful to create a new
    variable indicating the presence of missing data along with imputation.

Don’t use sigmoid or tanh

  • Sigmoid:

    • Formula:
    • Outputs values between 0 and 1.
    • Problems:
      • Vanishing gradients: For large positive or negative inputs, the gradient becomes very small, slowing down learning.
      • Outputs are not zero-centered: This can cause inefficient gradient updates.

  • Tanh:

    • Formula:
    • Outputs values between -1 and 1.
    • Problems:
      • Vanishing gradients: Like sigmoid, tanh saturates for large positive or negative inputs.
      • Slightly better than sigmoid because it’s zero-centered, but still inferior to ReLU and its variants.

  • Why avoid them:

    • Modern networks prefer ReLU or its variants because they avoid these issues, train faster, and perform better for deep architectures.

Use ReLU. Be careful with your learning rates

  • ReLU (Rectified Linear Unit):

    • ReLU is one of the most commonly used activation functions in deep learning because it introduces non-linearity while being computationally efficient.
    • Defined as: , it outputs the input directly if positive and zero otherwise.

  • Why ReLU is recommended:

    • Sparse activation: Only a subset of neurons are activated at any given time, making the network more efficient.
    • Avoids vanishing gradients: Unlike sigmoid or tanh, ReLU doesn’t squash gradients, which helps deeper networks learn effectively.

  • Caution about learning rates:

    • Problem: ReLU can cause the “dead neuron” problem, where some neurons output zero for all inputs and stop learning. This happens when the input to the ReLU is consistently negative due to high learning rates.
    • Solution: Use a moderate learning rate and techniques like proper weight initialization or adaptive optimizers (e.g., Adam).

Experiment with Leaky ReLU, Maxout, or ELU for improved performance or to avoid specific issues with ReLU.

Focus on incremental improvements when the model is already performing well.
